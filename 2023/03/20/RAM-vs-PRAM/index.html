RAM vs PRAM 随笔

Jing Zhou
2023-03-20
os

Intro

The algorithms required to solve a problem on a sequential computer is called a sequential algorithm. Algorithms written to solve a problem on a parallel computer are called parallel algorithms. The sequential algorithm is written in the form of steps to solve a problem. IN a parallel algorithm, we have to see where the communication is taking place, where we have to move the output of particular steps.

Algorithms written for a particular architecture cannot be applied to another architecture, in other words, to solve the same problem on different parallel computers, different algorithms must be written based parallel computer architecture.

writing parallel algorithm is a complex task, to deal with this problem various parallel models called abstract machine were designed hypothetically. Abstract machine are just imaginary notions that doesn’t exist in reality, including two types:

  • Random Access Machine (RAM)
  • Parallel Random Access Machine (PRAM)

这篇文章主要总结一下随机存取机和并行随机存取机的特征。

Random Access Machine (RAM)

随机存取机理论上有无限存储空间，每一个存储cell都有一个地址而且对其的任一操作（读取/写入，标准计算，布尔操作）都消耗一个同等的时间单元。RAM是一个标准的理想计算模型，即无限存储和存取操作的cost equal。

  1. The RAM model is a one address computer.
  2. RAM is a sequential machine. RAM consists of memory, read-only input tape, output tape only, and writes a program.
  3. The program is not stored in memory and cannot be modified.
  4. The input tape consists of a series of integers. Each time a input value is read, the input head moves forward on square.
  5. Similarly, the output head moves forward after each write. Memory consists of an infinite sequence of registers, designed r0, r1, r2,…
  6. Each register can contain an integer. Register r0 is the accumulator, where the calculation is done.
  7. RAM must have the same instructions as LOAD, STORE, READ, WRITE, ADD, SUBTRACT, MULTIPLY, DIVIDE, TEST, JUMP, and HALT.
  8. The worst-case time complexity function of a RAM program is f(n), which is the maximum time taken by the program to execute on all inputs of size n.
  9. We assume that each of these steps takes a constant, that is O(1) time.

In RAM, here each step of an algorithm consists of the following steps:

  • Read: (up to) N processors simultaneously (in parallel) read from N memory locations (as in memory) and store the values in their local registers.
  • Compute: (up to) N processors perform basic arithmetic or logical operations on values in their registers.
  • Write: (up to) N processors simultaneously write from their registers in (up to) N memory locations.

在PRAM中， 算法需要协调控制多个处理器的工作状态。

Parallel Random Access Machines (PRAM)

It helps to write a precursor parallel algorithm without any architecture constraints and also allows parallel-algorithm designers to treat processing power as unlimited. IT ignores the complexity of inter-process communication.

The following are the modules of which a PRAM consists of:

  1. A control unit, global memory, and an unbounded set of similar processors, each with its own private memory. 一个操控单元， 全局存储， 一组理想中无限数量的处理器，每个都有单独的存储（寄存器）。
  2. An active processor reads from global memory, performs required computation, and then writes to global memory. 一个处理器不断从全局存储读取，完成计算，然后写入全局存储。
  3. Therefore, if there are N processors in a PRAM, then N number of independent operations can be performed in a particular unit of time.

在存取共享memory的时候，经常会有处理器之间读写冲突，对此有以下常见的方案：

  • EREW: also called Exclusive Read Exclusive Write that doesn’t allow two processors to read and write from the same memory location at the same instance. 禁止多个处理器同时对相同的存储单元进行读写。
  • CREW: also called Concurrent Read Exclusive Write that allows all the processors to read from the same memory location but are not allowed to write into the same memory location at the same time. 允许并发读取但禁止同时写。
  • ERCW: also called Exclusive Read Concurrent Write that allows all the processors to write to the same memory but are now allowed to read the same memory location at the same time. 并发写但是禁止同时读。
  • CRCW: also called Concurrent Read Concurrent Write that allows both. 并发读，并发写。
