优先队列（priority Queue）

Jing Zhou
2022-08-27
Leetcode

Introduction

优先队列是队列的一种，跟队列不一样的是优先队列的出队顺序是按照优先级来的，在某些情况下，我们需要找到元素集合中的最小或者最大元素，比如在医院等待的病人按照紧急程度来治疗。常见的应用有：

  1. 数据压缩：赫夫曼编码算法
  2. 最短路径算法：Dijkstra算法
  3. 最小生成树算法：Prim算法
  4. 事件驱动仿真：顾客排队算法
  5. 选择问题：查找k个最值
  6. 排序问题：堆排序

优先队列经常用堆来实现，但是两者的定义并不对等。堆是一颗具特定性质的二叉树，堆的基本要求就是堆中所有节点的值必须大于或等于（大顶堆），或者必须小于或等于（小顶堆）。堆的另外一个性质是必须是完全二叉树。这些在移动窗口的算法题中有所介绍，此处不再深入讨论。

这篇博客的主要目的是整理一下Python中优先队列的实现，先是如何构造优先队列，然后怎么调用现成的包$heapq$。

Functions

优先队列是元素的容器，每个元素有一个相关的键值，其需要支持的主要操作有：

  • $insert(key, num)$，插入键值为$key$的数据到有限序列中，元素以其键值进行排序。键值也可以直接为下标。
  • $deleteMin/deleteMax$, 删除最小/最大值的元素
  • $getMin/getMax$，返回最小/大值但不删除。

根据需要一些常见的辅助操作有：

  • 第k最小/最大：返回队列中键值为第k个最下/最大的元素
  • 大小（size）：返回优先队列中的元素个数
  • 堆排序：基于键值的优先级将队列中的元素排序

Initialization

假设我们从一个数组开始构建一个大顶堆（通常排序问题用大顶堆比较多），当然数组可以为空，此时我们假设数组无序。

$self.heapify()$是堆化函数，即将无序数组构建成堆。我们假设元素的下表从$0$开始，那么我们只要从倒数第二层的，最后一个有子节点父节点往前遍历，把不符合要求的字节点换上来，这样可以保证大的元素从叶节点一直往上冒。该字节点的下表为$n//2-1$（可以画个示意图直观一点）：

现在来分析建堆的时间复杂度，可以直接画图来理解：
假设总高度为k，则从倒数第二层的节点开始，这一层的节点都要执行子节点比较然后交换（如果顺序是对的则不用交换）。如果进行了交换，要对更新了的那一支子树往下继续比较交换。那么在每一层的时间$s= 2^{i-1}*(k-i)$，其中$i$表示第几层，$2^{i-1}$表示这一层上有多少个元素，$k-i$表示最多需要的往下比较的次数。

举个例子，假如我的树一共$4$层，从第三层开始更新，每个节点只需要和其子节点进行$k-3=1$次比较和更换，第三层一共有$2^{3-1}=4$个节点，于是第三层需要操作数是$2^{3-1}*(4-3)$。于是迭代到第二层，每个节点最差的情况需要更新到枝节点，也就是$k-2=2$次而已。以此类推，总的时间计算为$S=2^{i-2}*1+2^{k-3}*2+…+2^{0}*(k-1)$。求解该等式可以先乘以2得到另一个等式相减得到：
$$S = 2^{k-1}+2^{k-2}+2^{k-3}…+2^{1}-(k-1)$$
等比数列求和大家都会了吧，直接代入求和公式：
$$S= a_1(1-q^n) \over (1-q)$$
得到：
$$S = 2^k-k-1$$ 其中$k$为完全二叉树的深度，所以$2^k \leq n < 2^k-1, log(n-1)<k \leq logn$，故综上所述$S=n-logn-1$，时间复杂度为$O(n)$。

Deletion

删除某一个元素也很简单，只要用末尾元素替代该元素的位置，同时末尾的元素要删掉，然后从该元素往下进行调整即可。最差情况是删除根节点（堆排序），时间复杂度是$O(logn)$：

可见删除一个元素的操作仅仅需要一次交换，一次弹出和层数次向下冒泡即可，总的操作为$O(1+1+Logn)=O(logn)$

堆排序

接下来是怎么用优先队列，第一个应用是堆排序:

  1. 对无序数组构建大顶堆，时间复杂度$O(n)$
  2. 此时队列的第一个元素一定是最大的，将其与队列最后一个元素对调位置，然后不考虑最后一个位置，对新的队首进行向下冒泡（即对首元素进行删除操作）。时间复杂度为$logn$。
  3. 往前同样的操作。

所以总的时间复杂度为$O(nlogn)$。

Python library

Python专用于优先队列的包叫$heapq$，非常方便，不过默认是小顶堆，所以通常想要大顶堆我们把数组转化为负数即可。$heapq$可以直接的对$list$数据进行操作，以下是全部methods：

  • $heappush(heap, item)$ 把一个数值$item$压入，并保持堆的性质（相当与上面的$insert()$函数），如果想要自己来也可以通过$arr.append(item)$和$heap.sort()$实现。
  • $heappop(heap)$，弹出最小值，如果只想读取可以用$heap[0]$。
  • $heappushpop(heap,item)$先压入一个，重新编排，然后弹出堆顶。比分步进行压入和弹出更高效
  • $heapify(x)$把数组转换成堆，线性时间复杂度$O(n)$，并且$in-place$（即空间复杂度$O(1)$）。
  • $heapreplace(heap,item)$，先弹出堆顶再压入，比分步更高效。

还有一些高阶的methods：
-$merge(*iterables,key=None,reverse=False)$，合并多个有序输入成一个有序输出（比如从多个$log$文件中合并时间戳），返回一个有序值的迭代器。$key$是排序的判断函数。类似$sorted(itertools.chaim(*iterables))$但是其返回的是迭代元素。
-$nlargest(n,iterable,key=None)$返回有一个列表，元素是$前n$个最大值，等同于$sorted(iterable,key=key,reverse=True)[:n]$。
-$nsmallest(n,iterable,key=None)$与上面类似。

这些函数的调用都是$heap.func()$格式。
